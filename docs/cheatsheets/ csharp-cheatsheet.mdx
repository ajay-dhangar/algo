This page serves as a high-performance, practical reference guide for C# patterns that appear frequently across Data Structures and Algorithms (DSA), competitive programming, technical interviews, and modern enterprise `.NET` development. 

## Language Fundamentals & Basics

C# is a strongly, statically typed object-oriented language. Every variable type must be declared explicitly or inferred clearly by the compiler at build time.

### Built-in Primitive Data Types

Knowing your data sizes prevents unexpected integer overflows during extreme test suite processing or mathematical additions.

```csharp title="Primitive Type Assignments"
// 1. Integer Types
int standardInt = 42;               // 32-bit signed integer (Range: ~ -2B to 2B)
long massiveInt = 100000000000L;    // 64-bit signed integer (Essential for heavy accumulation loops)

// 2. Floating-Point & High-Precision Types
float standardFloat = 3.14f;        // 32-bit single-precision floating point
double defaultDouble = 3.14159265;  // 64-bit double-precision floating point (Default decimal type)
decimal highPrecision = 99.99m;     // 128-bit high-precision value (Essential for financial math/currency)

// 3. Textual & Logical Types
char singleChar = 'A';              // 16-bit Unicode character unit
bool isConditionMet = true;         // Logical Boolean flag (true or false)
string textMessage = "Hello World"; // Immutable text character sequence object

```

### Operators & Logical Formats

```csharp title="Operational Expressions"
// Arithmetic Expressions
int addition = 5 + 3;
int division = 10 / 2;
double power = Math.Pow(3, 2);      // Exponentiation (3 squared)
```

## Continuous Data Structures

### Arrays (1D & 2D Matrices)

Arrays in C# occupy contiguous memory blocks, ensuring extremely fast index-based retrieval times ($O(1)$ complexity).

```csharp title="Array Instantiations"
int arraySize = 5;
int rowCount = 3;
int columnCount = 4;

// 1D Array Allocation (Defaults elements to 0)
int[] numbers = new int[arraySize];

// Multidimensional Rectangular Grid Allocation (Commonly used for graph matrices)
int[,] continuousGrid = new int[rowCount, columnCount];

// Jagged Array Allocation (Array of separate arrays with variable row lengths)
int[][] jaggedArray = new int[rowCount][];
for (int i = 0; i < rowCount; i++)
{
    jaggedArray[i] = new int[columnCount];
}

// Populating array values cleanly
Array.Fill(numbers, -1); // Rapid utility to initialize all index locations with a baseline value

```

### String Processing & StringBuilder

Because C# strings are **immutable objects**, modifying them in loops creates new instances behind the scenes, wasting memory. Use `StringBuilder` to modify textual strings efficiently.

```csharp title="String Manipulation Strategy"
string masterString = "abcdef";

char targetChar = masterString[2];          // String array indexing lookup -> 'c'
int textLength = masterString.Length;       // Returns element density -> 6
bool containsToken = masterString.Contains("cd"); // Check substring inclusion -> true
string slice = masterString.Substring(0, 3); // Slice parameters (startIndex, length) -> "abc"

// High-Efficiency Buffer Processing via StringBuilder
System.Text.StringBuilder buffer = new System.Text.StringBuilder();
buffer.Append("Initial Text ");
buffer.Append("Engineered Data");
buffer.Replace("Initial", "Processed");

string finalizedOutput = buffer.ToString(); // Converted cleanly to a formal string

```

### Lists (Dynamic Arrays)

Equivalent to `ArrayList` in Java or `std::vector` in C++. It resizes itself automatically as data grows.

```csharp title="Dynamic List Executions"
List<int> dynamicCollection = new List<int>();

dynamicCollection.Add(10);                // Append element to the tail -> [10]

int lookUpItem = dynamicCollection[0];    // Fast index access -> 10
dynamicCollection.RemoveAt(0);            // Evicts index tracking point -> [20]
int activeCount = dynamicCollection.Count; // Total tracking size metric

```

### Dictionaries & HashSets

Hash-table implementations that offer exceptionally fast average lookup and insertion speeds ($O(1)$ operations).

### Stacks, Queues, & Deques

Essential tools for managing elements in graph traversals like Breadth-First Search (BFS) and Depth-First Search (DFS).

```csharp title="Linear Storage Adaptations"
// 1. Stack Structure (LIFO - Last In First Out)
Stack<int> processingStack = new Stack<int>();
processingStack.Push(100);
int checkedTop = processingStack.Peek();  // Evaluates top layer without eviction
int evictedTop = processingStack.Pop();   // Removes and returns top node

// 2. Queue Structure (FIFO - First In First Out)
Queue<int> discoveryQueue = new Queue<int>();
discoveryQueue.Enqueue(50);
int structuralFront = discoveryQueue.Peek(); // Inspects item at front
int processedFront = discoveryQueue.Dequeue(); // Removes front item

// 3. Double-Ended Queue (Deque equivalent via LinkedList)
LinkedList<int> deque = new LinkedList<int>();
deque.AddFirst(1); // Add to front
deque.AddLast(2);  // Add to back
deque.RemoveFirst();
deque.RemoveLast();

```

### Priority Queues (Heaps)

Available natively in **modern .NET (Version 6+)**. You store an element along with a priority value to determine its processing order.

```csharp title="Priority Queue Implementation"
// Min-Heap Pattern: Smallest priority value floats to the top
PriorityQueue<string, int> minHeap = new PriorityQueue<string, int>();

minHeap.Enqueue("Low Priority Task", 10);
minHeap.Enqueue("Critical Hotfix Node", 1);

// Returns item with lowest numerical priority value first
string structuralNext = minHeap.Dequeue(); // Returns: "Critical Hotfix Node"

```

## Performance Tip: High-Speed I/O Blueprint

Standard `Console.ReadLine()` calls handle text encoding translations under the hood, which can slow down execution and cause Time Limit Exceeded (TLE) errors in competitive programming. This high-speed custom I/O template bypasses that overhead by parsing raw character byte streams directly from standard input.

```csharp title="Fast I/O Engine Pattern"
using System;
using System.IO;

public class FastIOEngine
{
    private static readonly Stream InputReader = Console.OpenStandardInput();
    private static readonly byte[] BufferWindow = new byte[1 << 16]; // 64KB input cache stream
    private static int _bufferHeadPointer = 0;
    private static int _totalBytesRead = 0;

    private static byte ReadNextByte()
    {
        if (_bufferHeadPointer >= _totalBytesRead)
        {
            _totalBytesRead = InputReader.Read(BufferWindow, 0, BufferWindow.Length);
            _bufferHeadPointer = 0;
            if (_totalBytesRead <= 0) return 0; // Return end of stream null tracking
        }
        return BufferWindow[_bufferHeadPointer++];
    }

    public static int ReadNextInteger()
    {
        int processingValue = 0;
        byte byteToken = ReadNextByte();

        while (byteToken <= 32)
        {
            if (byteToken == 0) return 0;
            byteToken = ReadNextByte();
        }

        bool isNegative = false;
        if (byteToken == '-')
        {
            isNegative = true;
            byteToken = ReadNextByte();
        }

        while (byteToken > 32)
        {
            if (byteToken >= '0' && byteToken <= '9')
            {
                processingValue = processingValue * 10 + (byteToken - '0');
            }
            byteToken = ReadNextByte();
        }

        return isNegative ? -processingValue : processingValue;
    }
}

```
